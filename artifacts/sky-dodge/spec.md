## Overview
Flappy Bird 스타일의 횡스크롤 비행기 게임. 스페이스바로 비행기를 조종해 파이프 장애물을 통과하며 점수를 올리고, 서버 리더보드에 기록을 남긴다.

## 시나리오

### 1. 게임 시작
[상황] 시작 화면이 표시된 상태에서
[동작] 'Play' 버튼을 클릭하면 게임 플레이 화면으로 전환되고 비행기가 화면 중앙에 나타난다

성공 기준:
- [ ] 시작 화면에서 'Play' 버튼 클릭 -> 게임 플레이 화면 전환, 비행기 표시
- [ ] 시작 화면에서 'Leaderboard' 버튼 클릭 -> 리더보드 상위 10명 표시

### 2. 비행기 조작
[상황] 게임 플레이 중일 때
[동작] 스페이스바를 누르면 비행기가 상승하고, 누르지 않으면 중력으로 하강한다

성공 기준:
- [ ] 스페이스바 누름 -> 비행기 Y 위치 상승
- [ ] 스페이스바 안 누름 -> 비행기 Y 위치 중력으로 하강
- [ ] 모바일에서 화면 탭 -> 스페이스바와 동일하게 상승

### 3. 파이프 장애물 통과
[상황] 게임 플레이 중 파이프가 오른쪽에서 왼쪽으로 스크롤될 때
[동작] 비행기가 상하 파이프 사이 틈을 통과하면 1점을 획득하고 점수가 화면에 표시된다

성공 기준:
- [ ] 파이프 틈 통과 -> 점수 1점 증가, 화면 상단에 현재 점수 표시
- [ ] 파이프는 오른쪽에서 생성되어 왼쪽으로 이동

### 4. 게임 오버 - 파이프 충돌
[상황] 게임 플레이 중일 때
[동작] 비행기가 파이프에 충돌하면 게임 오버 화면이 표시된다

성공 기준:
- [ ] 비행기가 파이프에 닿음 -> 게임 오버 화면 전환, 최종 점수 표시

### 5. 게임 오버 - 경계 충돌
[상황] 게임 플레이 중일 때
[동작] 비행기가 화면 상단 또는 하단 경계에 닿으면 게임 오버 화면이 표시된다

성공 기준:
- [ ] 비행기가 화면 상단에 닿음 -> 게임 오버 화면 전환
- [ ] 비행기가 화면 하단에 닿음 -> 게임 오버 화면 전환

### 6. 난이도 증가 - 속도
[상황] 게임 플레이가 진행될수록
[동작] 스크롤 속도가 점진적으로 증가하되 최대값을 초과하지 않는다

성공 기준:
- [ ] 시간 경과에 따라 파이프 스크롤 속도 증가
- [ ] 속도가 최대값에 도달하면 더 이상 증가하지 않음

### 7. 난이도 증가 - 파이프 틈 축소
[상황] 점수가 올라갈수록
[동작] 파이프 사이 틈이 점진적으로 좁아지되 최소값 이하로 줄어들지 않는다

성공 기준:
- [ ] 점수 증가에 따라 파이프 틈 크기 감소
- [ ] 틈 크기가 최소값에 도달하면 더 이상 감소하지 않음

### 8. 테마 변화
[상황] 게임 플레이 중 점수가 특정 구간에 도달하면
[동작] 배경 테마가 전환된다

성공 기준:
- [ ] 점수 0~9 -> 낮 테마 (밝은 하늘)
- [ ] 점수 10~24 -> 노을 테마
- [ ] 점수 25~49 -> 밤 테마 (별)
- [ ] 점수 50 이상 -> 우주 테마 (별, 행성)

### 9. 리더보드 점수 제출
[상황] 게임 오버 화면에서
[동작] 이름을 입력하고 'Submit' 버튼을 클릭하면 점수가 서버에 저장된다. 건너뛰기도 가능하다

성공 기준:
- [ ] 이름 입력 + 'Submit' 클릭 -> 서버에 점수 저장, 리더보드에 반영
- [ ] 'Skip' 클릭 -> 점수 제출 없이 시작 화면으로 이동
- [ ] 이름 미입력 시 'Submit' 비활성화

### 10. 프레임률 독립 물리 연산
[상황] 게임 플레이 중일 때
[동작] 물리 연산(중력, 이동)이 delta time 기반으로 동작하여 모니터 주사율(60Hz/120Hz)에 관계없이 동일한 게임 속도를 유지한다

성공 기준:
- [ ] 중력, 속도, 위치 업데이트가 delta time을 적용하여 프레임률 독립적으로 동작
- [ ] delta time 상한 클램핑(최대 50ms)으로 탭 복귀 시 비정상 동작 방지

### 11. 탭 비활성화 시 자동 일시정지
[상황] 게임 플레이 중 브라우저 탭이 비활성화되면
[동작] 게임이 자동으로 일시정지되고, 탭 복귀 시 일시정지 상태에서 재개할 수 있다

성공 기준:
- [ ] 탭 비활성화 -> 게임 자동 일시정지, 'Paused' 표시
- [ ] 탭 복귀 -> 일시정지 상태 유지, 스페이스바 또는 탭으로 재개

### 12. Canvas 고정 해상도
[상황] 게임 화면이 렌더링될 때
[동작] Canvas는 고정 해상도(480x640)로 렌더링되고, CSS 스케일링으로 화면에 맞춰 표시된다

성공 기준:
- [ ] Canvas 내부 해상도 480x640 고정
- [ ] 데스크톱/모바일 모두 비율 유지하며 화면에 맞춰 스케일링

### 13. 리더보드 조회
[상황] 시작 화면 또는 게임 오버 화면에서
[동작] 리더보드를 조회하면 상위 10명의 이름과 점수가 표시된다

성공 기준:
- [ ] 리더보드에 상위 10명의 이름, 점수, 순위 표시
- [ ] 점수 내림차순 정렬

## 범위

### 포함
- HTML5 Canvas 기반 게임 렌더링
- requestAnimationFrame 게임 루프
- 스페이스바/탭 조작
- 파이프 장애물 생성 및 충돌 판정
- 점수 시스템
- 복합 난이도 증가 (속도 + 틈 축소)
- 4단계 배경 테마 전환
- Next.js API Route 서버 리더보드
- 시작/게임플레이/게임오버 3화면 전환

### 제외
- 사운드/효과음: 첫 버전 범위 축소를 위해 제외. 추후 추가 가능
- 아이템 시스템 (코인, 쉴드 등): 핵심 게임 루프 완성 후 별도 기능으로 추가
- 소셜 공유: MVP 범위 외
- 사용자 인증: 리더보드는 이름만 입력하는 익명 방식

## 전제 조건
- Next.js 16 프로젝트 환경 구축 완료
- HTML5 Canvas를 지원하는 모던 브라우저
- bun 패키지 매니저 사용
- 물리 연산은 delta time 기반으로 프레임률 독립적 동작
- Canvas 고정 해상도 480x640, CSS 스케일링으로 반응형 대응
